# 10.01 Glossary

## Overview

The Glossary defines the core terminology used throughout the cmdOS documentation system.

Because cmdOS introduces a new AI-native computing model, many concepts require clear definitions to ensure consistency across:

- Architecture
- Development
- Product
- Business
- Research

The glossary provides a shared language for developers, researchers, contributors, and ecosystem participants.

Core principle:

Common Language

↓

Shared Understanding

↓

Better Development

↓

Scalable Ecosystem

---

# A

## Agent

An autonomous AI entity capable of understanding goals, creating plans, using capabilities, and executing tasks.

An Agent contains:

- Intelligence
- Memory
- Planning ability
- Execution capability

Example:

A Research Agent can collect information, analyze data, and generate reports.

---

## AI Execution

The process where AI systems perform real-world actions instead of only generating information.

Traditional AI:

Question

↓

Answer


AI Execution:

Intent

↓

Plan

↓

Action

↓

Result

---

## AI Native OS

An operating system designed around AI as the primary interaction layer.

Instead of users operating applications manually, users communicate intent and AI manages execution.

---

# C

## Capability

A capability is an executable ability that allows Agents to perform actions.

Examples:

- Send email
- Access database
- Control application
- Analyze data

Architecture:

Agent

↓

Capability

↓

Execution

---

## Capability Layer

The system layer responsible for providing executable actions to Agents.

It connects:

Agents

with:

Tools, APIs, Applications, and Services.

---

## Context

Information required for AI systems to understand situations correctly.

Examples:

- User information
- Task information
- Environment information
- Previous interactions

---

# D

## Desktop Agent

An AI Agent capable of interacting with desktop environments.

Capabilities:

- Application control
- File management
- Workflow automation
- System interaction

---

## Digital Worker

An AI-powered entity that performs tasks similar to a human employee.

Examples:

- Customer support Agent
- Research Agent
- Operations Agent

---

# E

## Execution Engine

The system responsible for converting AI decisions into real actions.

Responsibilities:

- Execute commands
- Manage workflows
- Monitor results
- Handle failures

---

## Execution Object Model

A core cmdOS concept defining tasks, actions, and workflows as structured executable objects.

Purpose:

Allow AI Agents to understand, manage, and execute operations reliably.

---

# I

## Intent

The user's desired outcome expressed naturally.

Example:

User:

"Prepare my weekly report."

Intent:

Create a business report automatically.

---

## Intent Understanding

The process where AI analyzes user requests and identifies:

- Goals
- Requirements
- Constraints
- Expected results

---

# K

## Kernel

The foundational layer responsible for managing core system operations.

In cmdOS:

Kernel provides:

- Execution control
- Resource management
- System coordination

---

# M

## Memory System

A system that stores and retrieves useful information for AI Agents.

Includes:

- User preferences
- Previous tasks
- Knowledge
- Context

---

## Marketplace

A platform where developers distribute:

- Agents
- Plugins
- Capabilities
- Workflows

---

# P

## Permission Gate

A security mechanism that requires user approval before sensitive actions are executed.

Examples:

- Sending messages
- Accessing private files
- Financial operations

---

## Plugin

An extension that adds new capabilities to cmdOS.

Examples:

- New integrations
- New workflows
- Specialized Agents

---

## Plugin Economy

The economic ecosystem created around developers building and monetizing cmdOS extensions.

---

# R

## Runtime

The environment responsible for running cmdOS components.

Responsibilities:

- Agent execution
- Task management
- Resource handling

---

## Router

A system responsible for selecting appropriate AI models or execution paths.

Example:

User Request

↓

AI Router

↓

Best Model Selection

↓

Response / Execution

---

# S

## SDK (Software Development Kit)

A collection of tools that allows developers to build on cmdOS.

Includes:

- Agent SDK
- Plugin SDK
- Capability SDK

---

## System Architecture

The overall structure defining how cmdOS components interact.

Includes:

- AI Layer
- Agent Layer
- Capability Layer
- Execution Layer

---

# T

## Task

A specific objective that cmdOS needs to complete.

Example:

"Send email to customer."

A task contains:

- Goal
- Required actions
- Execution plan

---

## Tool

An external resource that Agents can use to perform actions.

Examples:

- API
- Application
- Database
- Service

---

# W

## Workflow

A sequence of tasks and actions required to achieve a goal.

Example:

Marketing workflow:

Research

↓

Content Creation

↓

Approval

↓

Publishing

↓

Analysis

---

# AI Execution Operating System

The core identity of cmdOS.

Definition:

A computing system where humans express intent and AI Agents execute tasks across digital environments.

---

# cmdOS

An AI-native operating system designed to transform human intent into real-world execution.

Core model:

Intent

↓

Understanding

↓

Planning

↓

Execution

↓

Result

The Glossary provides a unified vocabulary for the cmdOS ecosystem.

As cmdOS evolves, new concepts and definitions can be added to maintain clarity across:

- Engineering
- Product development
- Business strategy
- Research

A shared language creates the foundation for a scalable AI Execution Operating System.
