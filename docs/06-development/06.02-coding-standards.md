# 06.02 Coding Standards

## Overview

The Coding Standards define the rules, conventions, and best practices used by developers when building cmdOS.

Because cmdOS is a large-scale AI Execution Operating System, consistent coding standards are required to maintain:

- Code quality
- System reliability
- Security
- Maintainability
- Team collaboration

Core principle:

Developer

↓

Coding Standards

↓

Source Code

↓

Review

↓

Production System

---

# 1. Coding Standards Purpose

The purpose of Coding Standards is to create a unified development approach across all cmdOS components.

It ensures:

- Consistent code style
- Easier maintenance
- Better collaboration
- Reduced technical debt
- Higher reliability

---

# 2. Coding Philosophy

cmdOS follows these coding principles:

## Simple

Code should be easy to understand.

Avoid:

- Unnecessary complexity
- Duplicate logic
- Over-engineering

---

## Modular

Code should be divided into independent components.

Example:

Kernel Module

↓

Runtime Module

↓

Agent Module

↓

Plugin Module

---

## Explicit

Code behavior should be clear.

Avoid:

- Hidden behavior
- Unexpected side effects
- Unclear dependencies

---

## Maintainable

Future developers should easily understand and modify the system.
