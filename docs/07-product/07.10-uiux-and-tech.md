# cmdOS UI/UX Direction & Technology Additions (v2)

For: cmdShell (Tauri app) and the cmdOS website.
Positioning: **The Operating System for AI Agents.**

---

# 1. Design Principle

The interface must say one thing: *"My agent is working on its own computer, and a
supervisor is watching it for me."* Not a chat window. Not a dashboard. A **command
deck** looking into a living machine.

Emotional target: the calm confidence of a control room — you see everything, you
approve what matters, you can undo the rest.

---

# 2. Visual Language

- **Base:** near-black `#0A0A0F`; panels `#12121A` with 1px `#1E1E2A` borders.
- **Accent:** electric cyan `#3DF5D0` (Alios / safe / go) + amber `#F5B23D` (needs approval)
  + red `#F5455C` (blocked / R3 / danger). Color *is* the safety signal.
- **Type:** monospace for system/agent output and the command field (JetBrains Mono);
  a clean sans (Inter) for human-facing labels and marketing.
- **Motion:** purposeful, not decorative — execution steps stream in; shadow forks
  visibly branch; promote animates the chosen future "snapping" into reality.
- **Texture:** subtle glass panels, faint scanline/grid, restrained glow on active state.
  Premium, quiet, futuristic. Never neon-overload, never corporate-SaaS-bright.

# 3. Core Screens

## 3.1 Onboarding
Sign in → **Create your agent**: name field, avatar picker (a gallery of distinct
agent faces + upload), personality/role selector (Assistant, Analyst, Operator…).
End: "Provisioning [Agent]'s computer…" with a booting-desktop animation.

## 3.2 The Deck (main screen)
Three zones:
- **Left rail:** your agents (avatars + trust score dot), Machines, history, settings.
- **Center — the Machine view:** the live streamed desktop of the agent's computer.
  A translucent overlay shows what the agent is doing right now (cursor trails,
  action labels). This is the hero surface — you literally watch your agent work.
- **Right — the Supervisor panel (Alios):** a live feed of Alios's judgments —
  each action with its R0–R3 badge, mandate check, and status
  (allowed / needs-approval / blocked). Approvals surface here as amber cards.

## 3.3 Choose Your Future
When the agent runs shadow forks: the Machine view splits into 2–3 thumbnails, each a
finished outcome with a diff summary ("14 files renamed", "3 emails drafted"). Big
Promote button on the chosen one; the rest fade out.

## 3.4 Approval Card (R2/R3)
Modal or amber card: what the action is, why it needs approval, the exact effect
(preview), Approve / Deny. R3 (irreversible) gets red framing and a confirmation.

## 3.5 The Ledger
A replayable timeline: every action, Alios's score, every intervention. Exportable.
The user's proof of exactly what happened.

---

# 4. Signature Interactions

- **Watch, don't read.** Progress is shown on the Machine, not narrated in chat.
- **Alios speaks only when it matters** — an approval, a warning, a blocked action.
  Silence = trust.
- **Trust score is ambient** — a small colored ring around each agent avatar;
  green calm, amber cautious, red quarantined. No numbers shouted at the user.
- **Undo is always one click** on any completed R0/R1 action in the ledger.

---

# 6. What to Avoid

- No chat-bubble-first layout (this is not a chatbot).
- No dense enterprise dashboard (this is not analytics software).
- No bright corporate palette.
- Never hide what the agent is doing — transparency is the product.
