# 07.00 Product Direction

## Overview

This document defines the core product direction of cmdOS.

cmdOS is a standalone AI-native operating environment where users delegate tasks to AI
Agents and let them execute real-world actions. It is not a chatbot, SaaS dashboard, or
decentralized application.

The primary product: **cmdOS Desktop Application**.

Core principle:

```
Human Intent → cmdOS → AI Agents → Computer Execution → Completed Work
```

---

# 1. Product Identity

- **Product name:** cmdOS
- **Positioning:** The AI Execution Operating System
- **Category:** AI-native operating system application

cmdOS creates a new interaction model between humans and computers.

```
Traditional:   Human → Application → Manual Operation → Result
cmdOS:         Human Intent → AI Understanding → Planning → Execution → Result
```

---

# 2. cmdOS Is NOT

- **A chatbot** — the purpose is execution, not answering questions.
- **A web application** — the primary experience is a dedicated desktop app, not a browser.
- **A DApp** — not a blockchain application. Blockchain, if used, is a supporting
  component only.
- **A SaaS dashboard** — the interface is an intelligent operating environment, not a set
  of management panels.

---

# 3. Primary Product

The first major product is a standalone desktop application for Windows, macOS, and Linux.
Its purpose is to provide an AI execution environment directly on the user's computer.

---

# 4. Product Architecture

```
cmdOS Desktop Interface (Control Center)
  ↓
Agent Runtime
  ↓
Execution Engine
  ↓
Capability Layer
  ↓
Operating System Environment
```

---

# 5. cmdOS Desktop Interface

The main human interaction surface. Users communicate through natural language, commands,
voice, and visual interaction.

Main functions: command input, agent status, execution visibility, permission requests,
task history, system control.

---

# 6. Control Center

## Definition

The Control Center manages AI Agents and lets users understand and control how cmdOS
operates. It is a surface within the Interaction Layer, not a separate runtime.

## Responsibilities

The Control Center manages:

- Agent lifecycle
- AI model configuration
- API connections
- Permissions
- Memory
- Execution logs
- Security settings

## User Experience

The user should feel: *"I have an AI system operating my computer,"* not *"I am chatting
with an assistant."*

---

# 7. Agent Runtime

The execution environment where AI Agents operate.

Responsibilities: receive tasks, understand objectives, create plans, select capabilities,
execute workflows, return results.

---

# 8. Execution Model

```
User Intent → Intent Understanding → Task Planning → Permission Check
→ Capability Selection → Execution → Verification → Result
```

---

# 9. Core Product Features

- **AI Command Center** — users describe outcomes ("Prepare my meeting tomorrow") and
  cmdOS checks the calendar, collects documents, drafts notes, requests approval, and
  completes the workflow.
- **Desktop Agent** — interacts with the computer environment: open applications, manage
  files, automate workflows, control software.
- **Memory System** — remembers useful context: user preferences, previous workflows,
  personal settings.
- **Permission System** — users keep control; sensitive actions (sending emails, deleting
  files, financial operations) require approval.

---

# 10. Product Evolution

Product evolution follows the canonical Stages in `docs/09-roadmap`:

- **Stage 1 — MVP** — personal AI execution on the desktop: individual productivity,
  basic workflows, desktop interaction.
- **Stage 2 — Agent Platform** — agent creation, plugins, developer ecosystem.
- **Stage 3 — Execution OS** — full computer control, multi-agent workflows,
  cross-platform execution.
- **Stage 4 — AI-Native OS** — autonomous AI workforce, enterprise AI operations, AI
  economy.

---

# 11. Product Principles

- **Intent First** — users describe outcomes, not commands.
- **Execution First** — value is completing tasks, not generating text.
- **Human Controlled** — AI assists; users keep authority.
- **Local First** — core execution runs on user devices whenever possible.
- **Transparent** — users can see what AI understands, plans, and executes.

---

# 12. Long-Term Vision

A world where humans no longer operate computers manually. Humans express intent, AI
Agents understand objectives, cmdOS executes the work, and the computer becomes an
intelligent execution partner.

---

# Summary

cmdOS is a standalone AI-native desktop operating system. It is not a chatbot, not a
DApp, and not a web dashboard.

```
Human Intent → cmdOS Desktop → Control Center → AI Agents → Execution Engine → Real-World Results
```

This direction is the foundation for building the AI Execution Operating System.
