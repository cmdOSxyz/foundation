# 1. Purpose

This document defines the Update Manager of cmdOS.

The Update Manager is the canonical subsystem responsible for securely distributing, validating, deploying and rolling back updates across every component of the cmdOS ecosystem.

It answers one question:

**"How can the platform evolve safely without compromising system integrity?"**

The Update Manager ensures that kernel components, plugins, capabilities, agents and runtime assets remain synchronized, verifiable and recoverable throughout their lifecycle.

---

# 2. Philosophy

Updates are verifiable.

Integrity precedes deployment.

Compatibility is mandatory.

Deployment is deterministic.

Rollback is always possible.

Every update is observable.
