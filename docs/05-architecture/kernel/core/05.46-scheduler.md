# 1. Purpose

This document defines the Scheduler of cmdOS.

The Scheduler is the canonical subsystem responsible for planning, prioritizing and executing time-based, event-driven and resource-aware tasks throughout the cmdOS ecosystem.

It answers one question:

**"When should work be executed?"**

The Scheduler provides deterministic timing, resource coordination and distributed scheduling for every execution environment.

---

# 2. Philosophy

Time is a first-class resource.

Scheduling is deterministic.

Execution follows priority.

Resources are utilized efficiently.

Scheduling survives restarts.

Every schedule is observable.

---
