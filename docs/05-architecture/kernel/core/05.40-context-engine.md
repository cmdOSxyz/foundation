# cmdOS Context
# Part 10.35.40 — Context Engine

Version: 1.0
Status: Stable
Category: AI Infrastructure

---

# 1. Purpose

This document defines the Context Engine of cmdOS.

The Context Engine is the canonical subsystem responsible for collecting, assembling, prioritizing, compressing, validating and distributing execution context across the cmdOS ecosystem.

It answers one question:

**"What information should the AI know before making a decision?"**

The Context Engine transforms fragmented information into a unified Execution Context for every AI request.

---

# 2. Philosophy

Context determines intelligence.

Information is assembled, not duplicated.

Every context source has measurable relevance.

Context is dynamic.

Execution receives only the minimum required information.

Context remains deterministic.

---

# 3. Canonical Principle

Every execution context follows

Request

↓

Context Discovery

↓

Source Collection

↓

Permission Verification

↓

Priority Ranking

↓

Deduplication

↓

Compression

↓

Validation

↓

Execution Context

↓

Distribution

Execution Context is immutable once delivered.

---

# 4. Responsibilities

The Context Engine is responsible for

- Context discovery
- Context collection
- Context aggregation
- Context prioritization
- Context compression
- Context validation
- Context deduplication
- Context distribution
- Context caching
- Context lifecycle

The Context Engine does not

- Execute AI inference
- Store permanent memory
- Select AI providers
- Execute workflows

- Source Independent

      ↓

# 20. Trust Boundary

Every context source crosses an explicit trust boundary.

Only validated, authorized and policy-compliant information becomes part of the Execution Context.

---

# 21. Architectural Invariants

1. Every Execution Context has a globally unique identifier.
2. Every context source is explicitly identified.
3. Permission validation precedes context distribution.
4. Context ranking is deterministic.
5. Compression preserves semantic intent.
6. Distributed contexts are immutable.
7. Unauthorized data is never included.
8. Context expiration is enforced.
9. Every context lifecycle is auditable.
10. Context assembly remains provider-independent.

---

# 22. Summary

The Context Engine is the canonical context orchestration subsystem of cmdOS.

By discovering, validating, prioritizing, compressing and distributing information from conversations, memories, workflows, capabilities and external systems, it constructs a unified Execution Context that enables every AI model, agent and workflow to operate with the right information at the right time while preserving security, efficiency, determinism and architectural consistency.
