# 1. Purpose

This document defines the complete Kernel Architecture of cmdOS.

It serves as the canonical architectural reference for every Kernel subsystem, runtime, service, capability, workflow, agent and execution model defined throughout the cmdOS specification.

It answers one question:

**"How does the entire cmdOS Kernel operate as one unified AI Execution Operating System?"**

This document unifies all previous Kernel specifications into a single coherent architecture.

---

# 2. Philosophy

The Kernel is deterministic.

Everything executes through the Kernel.

Everything is governed.

Everything is observable.

Everything is recoverable.

Everything is replaceable.

---

# 3. Architectural Vision

The cmdOS Kernel is an execution operating system rather than a conversational AI.

Its responsibilities are

- Interpret intent
- Build execution plans
- Coordinate intelligence
- Govern execution
- Protect system integrity
- Preserve state
- Recover from failure
- Continuously evolve

The Kernel never depends on any specific AI provider.
