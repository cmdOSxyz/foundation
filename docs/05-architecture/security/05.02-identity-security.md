# 05.02 Identity Security

## Overview

Identity Security defines how cmdOS identifies, verifies, and manages the trust of users, Agents, Plugins, Capabilities, and system components.

Identity is the foundation of security.

Before any operation can happen, cmdOS must know:

- Who is requesting the action
- What entity is performing the action
- What level of trust the entity has

# 3. Identity Types in cmdOS

cmdOS manages multiple identity types.

---

# User Identity

## Definition

Represents the human user operating cmdOS.

Contains:

- User identifier
- Authentication information
- Account information
- Security preferences

Purpose:

Determine who controls the system.

---

# Agent Identity

## Definition

Represents an AI Agent inside cmdOS.

Contains:

- Agent ID
- Agent type
- Owner information
- Trust level
- Execution scope

Example:

Trading Agent

Identity:

- Trading-agent-001
- Owner
- Allowed operations

---

# Plugin Identity

## Definition

Represents external extensions.

Contains:

- Plugin ID
- Developer identity
- Version
- Source information
- Trust status

Purpose:

Prevent unauthorized extensions.

---

# Capability Identity

## Definition

Represents executable abilities.

Contains:

- Capability ID
- Provider
- Version
- Permission requirements

Example:

Email Capability

---

# System Identity

## Definition

Represents internal cmdOS services.

Examples:

- Runtime Service
- Kernel Service
- Security Service
