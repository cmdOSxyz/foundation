# 05.02 Message System

## Overview

The Message System defines how cmdOS creates, sends, receives, routes, and processes structured messages between internal components.

The Message System is the foundation that allows Agents, Capabilities, Plugins, Runtime services, and external integrations to communicate reliably.

Core principle:

Source

↓

Message Creation

↓

Message Routing

↓

Message Processing

↓

Response

---

# 1. Message System Purpose

The purpose of the Message System is to provide a reliable communication mechanism inside cmdOS.

It enables:

- Component communication
- Agent coordination
- Service interaction
- Execution requests
- Result delivery

---

# 2. Position in Architecture

Message System belongs to the Communication Architecture layer.

Architecture flow:

Agent

↓

Message System

↓

Capability

↓

Runtime

↓

Kernel

↓

Execution

---

# 3. Message Definition

A Message is a structured information unit exchanged between components.

A Message contains:

- Source
- Destination
- Message type
- Payload
- Metadata
- Timestamp
- Security information

Example:

Message:

Source:

Trading Agent

Destination:

Trading Capability

Type:

Execution Request

Payload:

Trade parameters

---

# 4. Message Types

cmdOS supports different message categories.

---

# Command Message

## Purpose

Requests an action.

Examples:

- Execute task
- Start workflow
- Run capability

Flow:

Agent

↓

Command Message

↓

Capability

---

# Response Message

## Purpose

Returns execution results.

Examples:

- Success
- Failure
- Output data

Flow:

Capability

↓

Response Message

↓

Agent

---

# Event Message

## Purpose

Reports system changes.

Examples:

- Task completed
- Security alert
- Plugin installed

---

# Data Message

## Purpose

Transfers information.

Examples:

- Context data
- Memory data
- Execution records

---

# 5. Message Structure

A standard message contains:

## Header

Contains:

- Message ID
- Source
- Destination
- Timestamp
- Priority

---

## Payload

Contains actual information.

Examples:

- Commands
- Data
- Results

---

## Metadata

Contains additional information.

Examples:

- Version
- Context
- Security level

---

## Security Information

Contains:

- Identity
- Permissions
- Verification status

---
