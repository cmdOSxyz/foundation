# 05.01 Capability Overview

## Overview

The Capability Overview defines the fundamental role of Capabilities inside cmdOS.

Capabilities are the functional abilities that allow AI Agents to interact with the real world.

Without capabilities, Agents can understand and reason but cannot perform practical operations.

Core relationship:

AI Intelligence

↓

Agent

↓

Capability

↓

Action

↓

Runtime

↓

Kernel

↓

Real World Result

---

# 1. Capability Purpose

The purpose of Capabilities is to provide reusable execution abilities for cmdOS.

Capabilities allow the system to interact with:

- Applications
- Operating systems
- External services
- APIs
- Devices
- Business systems

Examples:

Email Capability:

- Read messages
- Send emails
- Manage communication


Desktop Capability:

- Open applications
- Control interfaces
- Perform automation


Trading Capability:

- Analyze markets
- Execute transactions
- Monitor positions

---

# 2. Capability Definition

A Capability is a modular system component that provides a specific ability.

A Capability defines:

- What operation is possible
- How the operation is accessed
- What permissions are required
- How execution is performed

Capability represents ability.

Permission represents authorization.

# Summary

Capability Overview defines how cmdOS gains practical abilities.

Agents decide what needs to be done.
