Version: 1.0

---

# 1. Purpose

This chapter defines the philosophical foundation of cmdOS.

While Chapter 01 explains why cmdOS exists, Chapter 02 defines how cmdOS should think.

Every architectural decision throughout the specification shall be consistent with the philosophy established here.

---

# 2. Philosophy as an Architectural Constraint

Architecture is not merely a technical design.

Architecture is the expression of a system's beliefs.

When technology evolves, implementation changes.

Philosophy remains.

For this reason, philosophy precedes architecture.

---

# 3. Core Philosophy

cmdOS is founded upon a single principle:

> Computing should adapt to human intent rather than requiring humans to adapt to computing.

Everything else derives from this statement.

# 5. Relationship to Architecture

The philosophy defined in this chapter governs every architectural layer.

Including:

- Intent Model
- Context Model
- Object Model
- State Model
- Identity
- Security
- Agents
- Kernel
- Desktop Runtime
- AI Execution Protocol

No architectural decision shall contradict the philosophy.

---

# 6. Relationship to Mission

Mission defines what cmdOS accomplishes.

Philosophy defines how cmdOS accomplishes it.

Both are equally authoritative.

---

# 7. Constitutional Authority

Chapter 02 has constitutional authority.

Every specification that follows inherits these principles.

Whenever two implementation approaches are technically valid, the approach that best aligns with this philosophy shall be preferred.
