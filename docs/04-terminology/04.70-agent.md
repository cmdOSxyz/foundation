# Agent

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Agent** within cmdOS.

Agent is the canonical autonomous intelligence entity responsible for understanding Objectives, performing Reasoning, creating Decisions, executing Tasks, and interacting with Tools to achieve desired outcomes.

Agents represent the intelligent operational actors of the AI Execution Operating System.

Every autonomous capability within cmdOS shall be performed through one or more Agents.

---

# 2. Definition

An **Agent** is a governed autonomous software entity capable of perceiving Context, performing Reasoning, making Decisions, executing Actions, and achieving Goals through available Capabilities.

An Agent provides:

- intelligence
- autonomy
- planning
- execution
- adaptation
- interaction

Agents transform Intent into coordinated execution.

---

# 3. Agent Is Not a Model

An AI Model provides intelligence computation.

An Agent uses Models together with Memory, Tools, Policies, and Execution capabilities.

```
Model

↓

Agent

↓

Execution
```

A Model generates intelligence.

An Agent applies intelligence.

---

# 4. Agent Is Not a Tool

A Tool performs a specific operation.

An Agent decides when and how Tools should be used.

```
Agent

↓

Tool Selection

↓

Action

↓

Execution
```

Agents orchestrate Tools.

---

# 5. Agent Is Not a Workflow

Workflow defines a structured process.

Agent creates, manages, or executes Workflows.

```
Agent

↓

Workflow

↓

Execution
```

Agents provide intelligence above workflows.

---
