# cmdOS Terminology

## Overview

The cmdOS Terminology layer defines the fundamental language and
concepts used throughout the cmdOS architecture.

cmdOS is an AI Execution Operating System.

Its purpose is to transform human intent expressed in natural language
into reliable, secure, and executable actions.

Core execution model:

Human Intent

↓

AI Understanding

↓

Planning

↓

Decision

↓

Action

↓

Execution

↓

Result

## 1. Purpose

The terminology layer creates a shared conceptual foundation for:

-   AI systems
-   Agents
-   Kernel
-   Runtime
-   Applications
-   Developers
-   Documentation

Without unified terminology, different components may interpret the same
concept differently.

## 2. cmdOS Mental Model

Traditional operating systems:

User → Application → Command → Execution

cmdOS:

User → Intent → Goal → Planning → Decision → Action → Execution → Result

The user describes desired outcomes.

cmdOS determines the execution process.

## 3. Terminology Domains

### Intent Domain

Defines how users express desired outcomes.

Concepts:

-   Intent
-   Intent Object
-   Intent Recognition
-   Intent Resolution

### Planning Domain

Defines how goals become execution strategies.

Concepts:

-   Goal
-   Plan
-   Task
-   Workflow
-   Dependency

### Decision Domain

Defines how cmdOS selects execution paths.

Concepts:

-   Decision
-   Reasoning
-   Selection
-   Optimization

### Execution Domain

Defines how actions are performed.

Concepts:

-   Action
-   Execution Object
-   Execution State
-   Result

### Agent Domain

Defines AI entities that perform tasks.

Concepts:

-   Agent
-   Capability
-   Memory
-   Runtime

### System Domain

Defines operating system concepts.

Concepts:

-   Kernel
-   Runtime
-   Resource
-   Service
-   Plugin

### Security Domain

Defines trust and control.

Concepts:

-   Identity
-   Permission
-   Policy
-   Verification
-   Audit

## 5. Design Principles

### Clear

Each concept has one primary meaning.

### Composable

Concepts can combine into complex workflows.

### Machine Understandable

Definitions must support AI reasoning and execution.

### Architecture Connected

Terminology must map to system architecture.
