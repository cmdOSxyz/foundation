# Scheduler

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Scheduler** within cmdOS.

Scheduler is the canonical coordination system responsible for determining when, where, and under what conditions Tasks, Actions, Workflows, Agents, and Executions should run.

Scheduler transforms planned work into controlled execution timing.

Every time-based, priority-based, or resource-aware execution within cmdOS shall be coordinated through one or more Scheduler components.

---

# 2. Definition

A **Scheduler** is a governed coordination component that assigns execution timing and resources to pending work.

A Scheduler determines:

- execution time
- execution order
- resource allocation
- priority handling
- dependency resolution
- workload distribution

Scheduler decides when work happens.

---

# 3. Scheduler Is Not a Queue

Queue stores pending work.

Scheduler decides when pending work should be processed.

```
Scheduler

↓

Queue

↓

Execution
```

Scheduler controls timing.

Queue controls availability.

---

# 4. Scheduler Is Not an Executor

Executor performs operations.

Scheduler only coordinates execution.

```
Scheduler

↓

Runtime

↓

Execution
```

Scheduler decides.

Runtime performs.

---

# 5. Scheduler Is Not a Workflow

Workflow defines process structure.

Scheduler determines when workflow components execute.

```
Workflow

↓

Scheduler

↓

Execution
```

---

# 28. Relationship to the Specification

Scheduler is the canonical execution coordination primitive within cmdOS.

Future specifications—including Scheduling Engine, Agent Scheduler, Resource Manager, Queue System, Runtime Controller, and Autonomous Execution Framework—shall define behavior according to the canonical Scheduler model established in this document.

Every controlled execution within cmdOS shall be coordinated through Scheduler mechanisms.

---

# 29. Summary

Scheduler provides the timing and coordination intelligence of cmdOS.

It connects Planning, Workflow, Tasks, Queues, Runtime, and Execution while maintaining efficiency, priority control, resource management, and governance.

By separating Scheduler from Queue, Executor, and Workflow, cmdOS establishes a scalable execution coordination layer capable of operating millions of autonomous AI processes.

Scheduler is therefore the canonical execution coordination primitive of the AI Execution Operating System.
