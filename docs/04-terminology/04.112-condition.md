# 1. Purpose

This document defines the canonical meaning of **Condition** within cmdOS.

Condition is the canonical evaluation input mechanism responsible for describing the circumstances, states, requirements, and environmental factors used by Rules, Policies, and Decision Systems to determine appropriate behavior.

Conditions allow cmdOS to understand the current situation before allowing, restricting, or modifying an operation.

Every Rule evaluation within cmdOS shall operate using one or more Conditions.

---

# 2. Definition

A **Condition** is a defined logical statement representing a specific state, requirement, or circumstance that can be evaluated by the system.

Condition defines:

- current state
- required state
- comparison criteria
- evaluation parameters
- expected values

Condition describes the situation.

# 33. Summary

Condition provides the evaluation input layer of cmdOS.

It connects Context, Rules, Policies, Decisions, Permissions, Actions, and Execution while enabling intelligent systems to understand circumstances before acting.

By separating Condition from Rule, State, and Context, cmdOS establishes a foundation for precise, explainable, and safe autonomous decision-making.

Condition is therefore the canonical evaluation input primitive of the AI Execution Operating System.
