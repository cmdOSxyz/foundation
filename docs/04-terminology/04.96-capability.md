# Capability

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Capability** within cmdOS.

Capability is the canonical representation of what an Entity, Agent, Plugin, Tool, or System is able to perform within the AI Execution Operating System.

Capabilities define available possibilities before Skills, Actions, and Execution are created.

Every executable intelligence component within cmdOS shall expose one or more Capabilities.

---

# 2. Definition

A **Capability** is a defined ability that allows an Entity to perform a specific class of operations, solve problems, or provide functionality.

Capability defines:

- what can be performed
- available functions
- supported operations
- required resources
- limitations
- execution conditions

Capability represents potential ability.

---

# 3. Capability Is Not Skill

Capability defines what an Entity is capable of doing.

Skill defines a specialized method for performing that capability.

```
Capability

↓

Skill

↓

Action

↓

Execution
```

Capability is ability.

Skill is expertise.

---

# 4. Capability Is Not Tool

Tool provides a mechanism to perform operations.

Capability describes the ability provided by that mechanism.

Example:

```
Tool:

Browser

↓

Capability:

Web Research
```

---

# 5. Capability Is Not Action

Action represents a specific operation.

Capability represents the ability to perform that operation.

```
Capability

↓

Possible Actions

↓

Execution
```

---

# 6. Capability Characteristics

Every Capability shall be:

- identifiable

# 30. Relationship to the Specification

Capability is the canonical ability representation primitive within cmdOS.

Future specifications—including Skill System, Plugin Architecture, Agent Marketplace, Tool Registry, and Extension Framework—shall define behavior according to the canonical Capability model established in this document.

Every Agent within cmdOS shall operate through available Capabilities.

---

# 31. Summary

Capability provides the ability layer of cmdOS.

It connects Agents, Skills, Tools, Actions, and Execution while allowing the system to discover, compose, and manage available abilities.

By separating Capability from Skill, Tool, and Action, cmdOS establishes a foundation for an expandable AI Operating System where intelligence can continuously gain new abilities.

Capability is therefore the canonical ability representation primitive of the AI Execution Operating System.
