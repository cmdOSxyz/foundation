# Resource

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Resource** within cmdOS.

Resource is the canonical representation of any asset, capability requirement, infrastructure component, or consumable element required for Agents, Tools, Actions, and Executions to operate.

Resources define what cmdOS can use, allocate, consume, and manage during operation.

Every Execution within cmdOS shall operate through controlled Resource management.

---

# 2. Definition

A **Resource** is any asset, capability, or available capacity that can be consumed, allocated, accessed, or utilized by an Entity during operation.

Resource defines:

- availability
- ownership
- capacity
- access rules
- usage limits
- cost
- lifecycle

Resource represents what enables execution.

---

# 3. Resource Is Not Tool

Tool performs operations.

Resource provides the required capacity or asset for those operations.

```
Resource

↓

Tool

↓

Action

↓

Execution
```

Example:

```
Resource:

GPU Compute

↓

Tool:

AI Model Runtime

↓

Action:

Generate Image
```

---

# 4. Resource Is Not Capability

Capability defines what can be done.

Resource defines what is required to perform it.

```
Capability

+

Resource

↓

Execution Ability
```

---

# 5. Resource Is Not Artifact

Artifact is created output.

Resource is consumed or utilized input.

```
Resource

↓

Execution

↓

Artifact
```

---

# 6. Resource Characteristics

Every Resource shall be:

- identifiable
- measurable
- allocatable
- permission-controlled
- observable
- governed
- manageable

Resources shall have defined ownership and limits.

---

# 7. Resource Identity

Every Resource shall possess a unique identifier.

Minimum metadata includes:

- Resource ID
- Type
- Owner
- Provider
- Capacity
- Availability
- Access Policy
- Cost Information
- Status

Identity enables resource management.

---

# 31. Summary

Resource provides the operational capacity layer of cmdOS.

It connects Capabilities, Tools, Actions, Execution, Cost Management, and Infrastructure while enabling AI systems to use available assets efficiently and safely.

By separating Resource from Tool, Capability, and Artifact, cmdOS establishes a foundation for scalable autonomous systems capable of managing compute, data, intelligence, and real-world assets.

Resource is therefore the canonical operational capacity primitive of the AI Execution Operating System.
