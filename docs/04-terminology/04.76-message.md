# Message

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Message** within cmdOS.

Message is the canonical communication data unit exchanged between Users, Agents, Services, Runtime Components, Tools, Plugins, and External Systems.

Messages enable structured communication across the cmdOS ecosystem.

Every communication between independent components within cmdOS shall be represented through one or more Messages.

---

# 2. Definition

A **Message** is a structured data object representing information transferred between two or more components.

A Message defines:

- sender
- receiver
- purpose
- content
- context
- metadata
- delivery requirements

Messages represent information exchange.

---

# 3. Message Is Not an Event

Event represents something that happened.

Message represents information being transmitted.

```
Event

↓

Message

↓

Consumer
```

An Event may be delivered through a Message.

---

# 4. Message Is Not an Action

Action performs an operation.

Message communicates information required to perform or understand operations.

```
Message

↓

Action Request

↓

Execution
```

Messages enable Actions.

---

# 5. Message Is Not a Command

Command is one type of Message requesting execution.

```
Message

├── Command

├── Event

├── Response

└── Notification
```

Every Command is a Message.

Not every Message is a Command.

---

# 6. Message Characteristics

Every Message shall be:

- identifiable
- structured
- traceable
- routable
- secure
- versioned
- observable

Messages shall preserve communication integrity.

---

# 7. Message Identity

Every Message shall possess a globally unique identifier.

Minimum metadata includes:

- Message ID
- Message Type
- Sender
- Receiver
- Timestamp
- Version
- Correlation ID
- Priority
- Status

Identity enables message tracking.

---

# 8. Message Lifecycle

Every Message progresses through:

```
Created

↓

Validated

↓

Sent

↓

Received

↓

Processed

↓

Completed

↓

Archived
```

Alternative states include:

- Failed
- Rejected
- Expired
- Retried

Lifecycle transitions shall remain observable.

# 30. Relationship to the Specification

Message is the canonical communication data primitive within cmdOS.

Future specifications—including Message Bus, Agent Communication Framework, Distributed Runtime, Event Architecture, Queue System, and Service Communication Layer—shall define behavior according to the canonical Message model established in this document.

Every component interaction within cmdOS shall be represented through Messages.

---

# 31. Summary

Message provides the communication foundation of cmdOS.

It connects Agents, Services, Runtime Components, Tools, Plugins, and External Systems through structured, secure, and observable information exchange.

By separating Messages from Events, Actions, and Commands, cmdOS establishes a scalable communication architecture capable of supporting distributed AI Agents, autonomous execution, and real-time system coordination.

Message is therefore the canonical communication data primitive of the AI Execution Operating System.
