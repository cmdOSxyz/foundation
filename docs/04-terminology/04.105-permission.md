# 1. Purpose

This document defines the canonical meaning of **Permission** within cmdOS.

Permission is the canonical authorization mechanism responsible for defining what an Entity is allowed to access, modify, execute, or control within the AI Execution Operating System.

Permission enables autonomous Agents to operate safely while maintaining user control, security boundaries, and system integrity.

Every Action and Execution within cmdOS shall operate under defined Permissions.

---

# 2. Definition

A **Permission** is a controlled authorization rule that defines whether an Entity may perform a specific operation on a specific Resource, Capability, Tool, or System.

Permission defines:

- allowed operations
- restricted operations
- access scope
- execution limits
- expiration conditions
- authority level

Permission defines what is allowed.

# 6. Permission Characteristics

Every Permission shall be:

- explicit
- controlled
- traceable
- revocable
- scoped
- temporary when required
- governed

Permissions shall never be unlimited by default.
