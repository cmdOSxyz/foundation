# Connector

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Connector** within cmdOS.

Connector is the canonical integration mechanism that enables cmdOS to communicate with external systems, services, applications, networks, and data sources.

Connectors provide controlled bridges between the AI Execution Operating System and the external world.

Every external interaction within cmdOS shall occur through one or more Connectors.

---

# 2. Definition

A **Connector** is a governed interface component that enables communication between cmdOS and an external system.

A Connector provides:

- authentication
- communication protocol
- data translation
- request handling
- response processing
- security enforcement

Connectors allow cmdOS to interact with external capabilities.

---

# 3. Connector Is Not a Tool

Tool performs an operation.

Connector provides the connection required for that operation.

```
Connector

↓

Tool

↓

Action

↓

Execution
```

Connector enables access.

Tool performs work.

---

# 4. Connector Is Not an API

API is an external interface exposed by another system.

Connector is cmdOS's managed integration layer for accessing that API.

```
External API

↓

Connector

↓

cmdOS Capability
```

Connectors abstract external complexity.

---

# 5. Connector Is Not a Plugin

Plugin extends cmdOS functionality.

Connector connects cmdOS to external systems.

```
Plugin

↓

Connector

↓

External System
```

A Plugin may contain one or more Connectors.

---

# 29. Summary

Connector is the bridge that allows cmdOS to interact with the external world.

It connects Applications, APIs, Blockchains, Devices, Databases, and Services with Agents, Skills, Tools, and Executions while maintaining security, abstraction, and governance.

By separating Connectors from APIs, Tools, and Plugins, cmdOS establishes a scalable integration architecture capable of connecting an AI operating system with the entire digital ecosystem.

Connector is therefore the canonical external integration primitive of the AI Execution Operating System.
