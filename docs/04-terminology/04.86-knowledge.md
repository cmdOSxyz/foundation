# Knowledge

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Knowledge** within cmdOS.

Knowledge is the canonical intelligence structure responsible for organizing, connecting, and representing meaningful information that enables Agents and Systems to understand concepts, reason about situations, and make informed Decisions.

Knowledge transforms raw information and experiences into usable intelligence.

Every intelligent reasoning capability within cmdOS shall depend on one or more Knowledge systems.

---

# 2. Definition

**Knowledge** is a structured representation of understood information, relationships, concepts, rules, and patterns that can be used for Reasoning and Decision making.

Knowledge defines:

- concepts
- relationships
- facts
- rules
- patterns
- explanations
- context

Knowledge represents understanding.

---

# 3. Knowledge Is Not Data

Data represents raw information.

Knowledge represents interpreted and structured meaning.

```
Data

↓

Information

↓

Knowledge
```

Data becomes useful when transformed into Knowledge.

---

# 4. Knowledge Is Not Memory

Memory stores retained experiences and information.

Knowledge stores structured understanding.

```
Memory

↓

Knowledge

↓

Reasoning
```

Memory provides experience.

Knowledge provides understanding.

---

# 5. Knowledge Is Not Intelligence

Knowledge provides information structures.

Intelligence uses Knowledge to reason and act.

```
Knowledge

+

Reasoning

↓

Intelligence
```
