# 04.00 Canonical Glossary

Canonical terms for cmdOS. Deprecated aliases must be replaced on the next edit of any
affected document. A CI check (`tools/ci/check-docs.sh`) fails the build if a deprecated
alias appears.

---

## Terms

| Canonical term | Meaning | Deprecated aliases |
|----------------|---------|--------------------|
| Control Center | User-facing management/admin surface (agents, models, connections, permissions, memory, logs, security). A surface within the Interaction Layer. | Admin Desktop, Admin Control Center |
| Interaction Layer | Input/UI surface that captures user requests. | — |
| Intent | Structured, validated artifact produced from a user request. Not a stack layer. | "Intent Layer" (used as a UI name) |
| Intelligence Layer | AI cognition / reasoning / routing layer. | AI Cognition (as a layer name) |
| Agent Runtime | Environment where Agents execute. | — |
| Execution Engine | Kernel component that runs execution plans. | — |
| Capability | Executable ability contract + implementation invoked by an Agent. | — |
| Plugin | Distribution package that registers Capabilities/Agents after validation. | — |
| Marketplace | Single distribution channel for Plugins. | Plugin Marketplace, Capability Marketplace |

---

## Canonical stack

```
Interaction Layer → Intelligence Layer → Agent Layer → Capability Layer
→ Communication Layer → Runtime Layer → [Security: cross-cutting] → Kernel
```

Notes:

- "Admin Runtime" is **removed** as a stack layer. It conflated the Control Center
  (a UI surface) with the Agent Runtime (an execution layer).
- The **Control Center** is a surface within the Interaction Layer, not a separate runtime.
- Agents invoke **Capabilities**, never **Plugins**.
- There is exactly one **Marketplace**, and it distributes **Plugins**.
