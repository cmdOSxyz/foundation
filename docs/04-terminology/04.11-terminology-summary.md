# 04.11 Terminology Summary

## Overview

This document provides the complete terminology summary of cmdOS.

It connects all core concepts into one unified execution model.

The purpose is to give humans, AI systems, and developers a shared understanding of how cmdOS transforms human intent into machine execution.

Core principle:

Human Intent

↓

Machine Execution

---

# 1. Complete cmdOS Execution Model

The complete execution pipeline:

Intent

↓

Goal

↓

Planning

↓

Decision

↓

Action

↓

Execution

↓

Result


Each stage has a specific responsibility.

---

# 2. Intent

## Meaning

Intent represents what the user wants to achieve.

Intent is expressed naturally through human language.

Example:

"Create a report from my sales data."

Intent describes the desired outcome, not technical instructions.

---

# 3. Goal

## Meaning

Goal represents the target state that should be achieved.

Intent:

"Create a report."

Goal:

"Generate a completed sales report."

Goal defines success.

---

# 4. Planning

## Meaning

Planning converts goals into structured strategies.

Planning determines:

- Required steps
- Dependencies
- Resources
- Execution order

Goal becomes a plan.

---

# 5. Decision

## Meaning

Decision selects the best execution path.

Decision determines:

- Which capability to use
- Which agent should execute
- Which resources are required
- Which strategy is optimal

---

# 6. Action

## Meaning

Action represents an executable operation.

Examples:

- Send message
- Create file
- Call API
- Execute workflow step

Actions are the smallest execution units.

---

# 7. Execution

## Meaning

Execution performs approved actions through cmdOS infrastructure.

Execution manages:

- Execution state
- Resource usage
- Errors
- Results

Execution is controlled by the Kernel.

---

# 8. Result

## Meaning

Result represents the outcome of execution.

Results may include:

- Success
- Failure
- Partial completion
- Required confirmation

---

# 9. Agent

## Meaning

Agent is an AI execution entity.

Agents provide:

- Understanding
- Reasoning
- Planning
- Capability selection

Agents transform intelligence into executable operations.

---

# 10. Capability

## Meaning

Capability represents what the system can do.

Examples:

- Email
- Calendar
- Browser
- Trading
- File management

Capability defines ability.

Permission defines access.

---

# 11. Runtime

## Meaning

Runtime provides the environment where Agents and workflows operate.

Runtime manages:

- Active processes
- Context
- Lifecycle
- Coordination

---

# 12. Kernel

## Meaning

Kernel is the execution authority of cmdOS.

Kernel controls:

- Execution
- Resources
- Security
- System state

Agents decide.

Kernel executes.

---

# 13. Security

## Meaning

Security ensures all execution is controlled and trusted.

Security manages:

- Identity
- Authentication
- Permission
- Policy
- Authorization
- Audit

---
