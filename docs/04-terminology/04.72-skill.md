# Skill

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Skill** within cmdOS.

Skill is the canonical representation of a reusable capability package that enables an Agent, System, or Actor to perform a specific category of operations.

Skills define what an Agent is capable of doing and provide a modular mechanism for expanding intelligence and execution abilities.

Every advanced Agent capability within cmdOS shall be represented through one or more Skills.

---

# 2. Definition

A **Skill** is a governed capability module that provides an Agent with the knowledge, procedures, Tools, and behaviors required to perform a specific function.

A Skill defines:

- capability scope
- required knowledge
- required Tools
- execution patterns
- permissions
- usage constraints
- performance criteria

Skills represent learned or installed abilities.

---

# 3. Skill Is Not a Capability

Capability represents the possibility of performing an ability.

Skill represents the packaged implementation of that ability.

```
Capability

↓

Skill

↓

Action

↓

Execution
```

Capability defines possibility.

Skill enables practical usage.

---

# 4. Skill Is Not a Tool

Tool performs an operation.

Skill combines multiple elements to achieve a higher-level ability.

```
Skill

├── Knowledge

├── Instructions

├── Tools

├── Workflows

└── Policies
```

A Skill may use multiple Tools.

---

# 5. Skill Is Not an Agent

Agent is an autonomous intelligence entity.

Skill is an ability that an Agent can possess.

```
Agent

+

Skills

↓

Agent Capability
```

Agents use Skills.

Skills do not operate independently unless assigned.

---

# 6. Skill Characteristics

# 27. Skill Invariants

The following statements shall always remain true.

1. Every Skill has a unique identity.
2. Every Skill has defined capabilities.
3. Every Skill has controlled permissions.
4. Every Skill belongs to an owner or provider.
5. Every Skill execution is traceable.
6. Every Skill version is preserved.
7. Skills cannot bypass Policies.
8. Skills cannot exceed assigned authority.

These invariants apply throughout the cmdOS platform.

# 28. Relationship to the Specification

Skill is the canonical Agent capability expansion primitive within cmdOS.

Future specifications—including Skill Registry, Agent Marketplace, Training System, Capability Framework, Plugin System, Agent Learning Architecture, and Community Ecosystem—shall define behavior according to the canonical Skill model established in this document.

Every expandable Agent ability within cmdOS shall be represented through one or more Skills.

---

# 29. Summary

Skill is the modular capability layer that allows Agents to expand, specialize, and evolve.

It connects Agents with Capabilities, Tools, Knowledge, Workflows, and Execution while maintaining security, versioning, and governance.

By separating Skills from Agents, Tools, and Capabilities, cmdOS establishes an ecosystem where AI workers can continuously gain new abilities through installation, learning, and collaboration.

Skill is therefore the canonical capability module primitive of the AI Execution Operating System.
