# Action

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **Action** within cmdOS.

Action is the canonical representation of a specific operation that an Agent, User, or System intends to perform after a Decision has been made.

Actions transform Plans and Decisions into executable operations.

Every execution process within cmdOS shall begin with one or more Actions.

---

# 2. Definition

An **Action** is a defined operation that represents an intended change, interaction, or activity performed by an Entity through available Tools and Capabilities.

An Action defines:

- operation to perform
- target object
- required Tool
- input parameters
- expected outcome
- execution conditions

Action represents what will be performed.

---

# 3. Action Is Not Decision

Decision determines what should happen.

Action defines the specific operation to perform.

```
Decision

↓

Action

↓

Execution
```

Decision chooses.

Action performs.

---

# 4. Action Is Not Tool

Tool provides the mechanism.

Action represents the specific usage of that mechanism.

```
Tool

↓

Action

↓

Execution
```

Example:

```
Tool:

Browser

↓

Action:

Open Website
```

---

# 5. Action Is Not Execution

Action defines the intended operation.

Execution is the actual process of carrying out that operation.

```
Action

↓

Execution

↓

Result
```

---

# 6. Action Characteristics

Every Action shall be:

- identifiable
- executable
- traceable
- permission-controlled
- observable
- verifiable
- reversible when possible

Actions represent controlled intent to operate.

---

# 7. Action Identity

Every Action shall possess a unique identifier.

Minimum metadata includes:

- Action ID
- Source Decision
- Agent Identity
- Tool Reference
- Target Object
- Parameters
- Permissions
- Timestamp
- Status

Identity enables action tracking.

---

# 8. Action Lifecycle

Every Action progresses through:

```
Created

↓

Validated

↓

Authorized

↓

Scheduled

↓

Executing

↓

Completed

↓

Verified

↓

Archived
```

Alternative states include:

- Rejected
- Failed
- Cancelled
- Rolled Back

- create file
- execute command
- manage process

---

## Communication Action

Performs communication operations.

Examples:

- send message
- send email
- create notification

---

## Data Action

Manipulates information.

Examples:

- retrieve data
- transform data
- store information

---

## Application Action

Controls applications.

Examples:

- open application
- interact with UI
- perform workflow

---

## Blockchain Action

Performs decentralized operations.

Examples:

- submit transaction
- query blockchain
- interact with contract

---

## Development Action

Supports software engineering.

Examples:

- modify code
- run tests
- deploy application

# 30. Action Invariants

The following statements shall always remain true.

1. Every Action has an identity.
2. Every Action has a source.
3. Every Action requires authorization.
4. Every Action uses defined capabilities.
5. Every Action execution is traceable.
6. Every Action follows Governance.
7. Every Action result can be verified.
8. Actions cannot execute without permission.

---

# 31. Relationship to the Specification

Action is the canonical operation representation primitive within cmdOS.

Future specifications—including Execution Engine, Transaction System, Tool Runtime, Workflow Engine, and Agent Runtime—shall define behavior according to the canonical Action model established in this document.

Every real-world operation performed by cmdOS shall be represented as an Action.

---

# 32. Summary

Action provides the operation layer of cmdOS.

It connects Decisions, Plans, Tools, Transactions, Execution, and Results while allowing AI Agents to safely transform intelligence into real-world operations.

By separating Action from Decision, Tool, and Execution, cmdOS establishes a foundation for controlled autonomous systems capable of performing meaningful tasks.

Action is therefore the canonical operation representation primitive of the AI Execution Operating System.
