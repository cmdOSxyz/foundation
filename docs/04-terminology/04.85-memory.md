# 1. Purpose

This document defines the canonical meaning of **Memory** within cmdOS.

Memory is the canonical information retention system responsible for storing, organizing, retrieving, and managing experiences, Context, knowledge, preferences, and historical information required for intelligent operation.

Memory enables Agents and Systems within cmdOS to maintain continuity, improve performance, and make informed Decisions based on previous experiences.

Every persistent intelligence capability within cmdOS shall rely on one or more Memory systems.

---

# 2. Definition

A **Memory** is a governed information storage representation that preserves information required for future understanding, reasoning, Decision making, and execution.

Memory stores:

- experiences
- Context
- knowledge
- preferences
- history
- learned patterns
- relationships

Memory allows intelligence to persist over time.

# 9. Memory Types

cmdOS recognizes multiple Memory categories.

---

## Short-Term Memory

Stores temporary information.

Examples:

- current conversation
- active Context
- temporary Tasks

Purpose:

Immediate reasoning support.

---

## Long-Term Memory

Stores persistent information.

Examples:

- user preferences
- historical interactions
- learned behavior

Purpose:

Long-term continuity.

---

## Episodic Memory

Stores experiences.

Examples:

- previous executions
- completed workflows
- past events

Purpose:

Learning from experience.

---

## Semantic Memory

Stores structured knowledge.

Examples:

- concepts
- relationships
- facts

Purpose:

Understanding and reasoning.

---

## Procedural Memory

Stores methods and procedures.

Examples:

- workflows
- strategies
- execution patterns

Purpose:

Knowing how to perform tasks.
