# State

Version: 1.0  
Status: Canonical  
Category: Foundation

---

# 1. Purpose

This document defines the canonical meaning of **State** within cmdOS.

State is the canonical representation of the current condition, status, and configuration of an Entity, Process, Agent, System, Workflow, or Execution at a specific point in time.

State enables cmdOS to understand the current situation, determine possible transitions, and control future operations.

Every dynamic component within cmdOS shall maintain one or more State representations.

---

# 2. Definition

A **State** is a structured representation of the current condition of an Entity or Process.

State defines:

- current condition
- active properties
- available capabilities
- current progress
- relationships
- allowed transitions
- operational status

State represents where something currently is.

---

# 3. State Is Not Context

State represents current condition.

Context includes State plus additional surrounding information.

```
State

+

Environment

+

History

+

Relationships

↓

Context
```

State is one component of Context.

---

# 4. State Is Not Memory

State represents the present.

Memory represents retained information from the past.

```
Current State

↓

Experience

↓

Memory
```

State changes over time.

Memory preserves history.

---

# 5. State Is Not Event

Event represents a change occurrence.

State represents the result after changes.

```
Event

↓

State Change

↓

New State
```

---

# 6. State Characteristics

Every State shall be:

- identifiable
- observable
- consistent
- traceable
- versioned
- governed
- transitionable

State shall accurately represent current reality.

---

# 7. State Identity

Every State instance shall possess a unique identifier.

Minimum metadata includes:

- State ID
- Entity Reference
- State Type
- Version
- Timestamp
- Owner
- Status
- Previous State Reference

Identity enables state tracking.

# 29. Relationship to the Specification

State is the canonical condition representation primitive within cmdOS.

---

# 30. Summary

State provides the condition awareness layer of cmdOS.

It connects Entities, Events, Actions, Decisions, Transactions, and Executions while allowing the system to understand current conditions and control transitions.

By separating State from Context, Memory, and Events, cmdOS establishes the foundation for reliable autonomous systems capable of maintaining continuity and controlled evolution.

State is therefore the canonical condition representation primitive of the AI Execution Operating System.
